# mal-ko

**mal-ko** — агрегатор данных об угрозах (threat-intel aggregator) для защиты
собственных систем. Инструмент собирает данные об уязвимостях и вредоносном ПО
из открытых источников и проверяет по ним:

- **зависимости проекта** (`requirements.txt`, `package.json`) — по базе OSV.dev
  с перекрёстной проверкой найденных CVE по каталогу CISA KEV;
- **установленное ПО** на компьютере — поиск по NVD и проверка по CISA KEV;
- **файлы** — по SHA-256-хэшам в базе MalwareBazaar (abuse.ch) плюс
  **эвристический статический анализ** (PE-заголовки, энтропия секций,
  подозрительные импорты и скрипты);
- **точки автозапуска** (`scan-persist`) — Run-ключи реестра, папка
  «Автозагрузка», планировщик задач (только обнаружение, ничего не удаляет);
- **новые и изменённые файлы в реальном времени** — монитор `monitor`
  отслеживает каталоги и отправляет известную малварь в карантин.

Инструмент полностью защитный (defensive): никакого эксплойт-кода, только
запросы к открытым базам данных и статический анализ. Написан на Python 3.10+,
использует **только стандартную библиотеку** — ничего устанавливать через pip
не нужно. Доступна также сборка standalone `.exe` (см. ниже).

## Это НЕ антивирус

Честное предупреждение: mal-ko — это сканер и простой сторож по открытым базам
угроз, а не полноценный антивирус. У него **нет**:

- драйвера уровня ядра и перехвата системных вызовов;
- поведенческого анализа, песочницы и сигнатурного движка уровня настоящего AV;
- защиты сети, памяти и загрузки системы;
- автоматического «лечения» — только карантин файлов, подтверждённых по хэшам.

Встроенная эвристика расширяет покрытие (упакованные PE, подозрительные
скрипты, двойные расширения), но это **простые статические правила**: они дают
ложные срабатывания на легальном ПО и пропускают неизвестную малварь без
известных признаков. Режим реального времени (`monitor`) опрашивает каталоги
по таймеру (polling), а не перехватывает события файловой системы, поэтому
очень быстрые атаки (создал-запустил-удалил между опросами) он может
пропустить. mal-ko не заменяет полноценный антивирус — это дополнительный
слой защиты для ваших собственных систем.

## Требования и запуск

- Python 3.10 или новее. Зависимостей нет.
- Клонируйте репозиторий и запускайте из его корня:

```bash
python -m malko --help
```

## Подкоманды

### scan-deps — проверка зависимостей

Находит `requirements.txt` (PyPI) и `package.json` (npm, секции `dependencies`
и `devDependencies`) в указанном каталоге (по умолчанию — текущий), приводит
semver-диапазоны (`^`, `~`, `>=`) к конкретной версии и проверяет каждый пакет
по OSV.dev. Найденные CVE дополнительно сверяются с каталогом CISA KEV —
совпадения помечаются как **ACTIVELY EXPLOITED**.

```bash
python -m malko scan-deps ./my-project
python -m malko scan-deps ./my-project --json report.json
```

Проверяются только пакеты с точной версией (`name==1.2.3`); неразрешимые
записи пропускаются с предупреждением.

### scan-system — проверка установленного ПО

Составляет инвентарь установленных программ (Windows: ветки реестра
`...\Uninstall` в HKLM/HKCU; Linux: `dpkg-query`, при отсутствии — `rpm`;
иначе выдаёт сообщение о неподдерживаемой системе). Для каждого продукта
выполняется ключевой поиск по NVD (с паузой ~1,5 с между запросами — лимит
NVD без API-ключа) и проверка по полям vendor/product каталога CISA KEV.

```bash
python -m malko scan-system
python -m malko scan-system --max-products 10 --json system.json
```

`--max-products N` (по умолчанию 30) ограничивает число проверяемых продуктов,
чтобы сканирование не длилось слишком долго.

### scan-files — проверка файлов по хэшам

Считает SHA-256 для каждого файла (каталоги обходятся рекурсивно) и проверяет
хэш в MalwareBazaar. Статус `hash_not_found` означает «в базе нет»
(чистый/неизвестный).

```bash
python -m malko scan-files ~/Downloads
python -m malko scan-files file1.exe ./some-dir --json files.json
python -m malko scan-files ~/Downloads --quarantine   # находки — сразу в карантин
```

> Внимание: API MalwareBazaar теперь требует бесплатный ключ (Auth-Key).
> Получите его на https://abuse.ch/ и задайте переменную окружения:
>
> ```bash
> export MB_AUTH_KEY="ваш-ключ"
> python -m malko scan-files ~/Downloads
> ```
>
> Без ключа сканирование не прерывается: выдаётся предупреждение, и файлы
> проверяются только эвристикой.

## Эвристический анализ

Поверх проверки по хэшам работает эвристический движок (`scan-files`,
`monitor`, `scan-persist`; отключается флагом `--no-heuristics`). Он
начисляет баллы за статические признаки: **≥ 50 — SUSPICIOUS**,
**≥ 90 — SUSPICIOUS+**. Что ловят правила:

- **PE-анализ** (ручной разбор заголовков): секции с высокой энтропией
  (> 7.2 — вероятно, упакован), комбинация импортов
  `VirtualAlloc + WriteProcessMemory + CreateRemoteThread` (инъекция в
  процесс), `URLDownloadToFile`, `WinExec`, `ShellExecute`,
  `IsDebuggerPresent`, `VirtualProtect`, `SetWindowsHookEx`, импорты только
  через `LoadLibrary/GetProcAddress`, отсутствие таблицы импортов у `.exe`,
  точка входа в последней секции, секция одновременно записываемая и
  исполняемая;
- **двойные расширения** (`photo.jpg.exe`, `invoice.pdf.scr`);
- **исполняемые файлы и скрипты в рискованных местах** (Temp, Downloads);
- **тестовая строка EICAR** (вердикт `EICAR-TEST`, обрабатывается как
  подтверждённая угроза — карантинируется);
- **макросы Office**: `AutoOpen`/`Auto_Open`, `WScript.Shell` в legacy
  `.doc`/`.xls`, VBA-проект в `.docm`/`.xlsm`;
- **подозрительные скрипты** (ps1/bat/vbs/js): `powershell -enc`,
  `FromBase64String`, `Invoke-Expression`, `certutil -decode`, `mshta http`,
  `DownloadString`.

**Важно про ложные срабатывания.** Эвристика — это подозрения, а не
доказательство: легальные браузеры, игры и драйверы регулярно набирают
SUSPICIOUS-баллы (проверено на практике: Firefox, Opera, Roblox). Поэтому
подозрительные файлы **не перемещаются в карантин автоматически** — только
громкое предупреждение, событие в журнале и запись в находках (фиолетовый
бейдж SUSPICIOUS в веб-панели). В карантин попадают только подтверждённые
совпадения по блоклисту и EICAR.

### scan-persist — проверка точек автозапуска

Перечисляет места закрепления в системе и проверяет каждую цель по блоклисту
и эвристике (Windows: Run/RunOnce в HKCU и HKLM, папка «Автозагрузка»,
задачи планировщика через `schtasks`; Linux: `~/.config/autostart/*.desktop`
и `crontab -l`). Сообщает о записях, чья цель отсутствует на диске, лежит во
временной/пользовательской папке, подозрительна по эвристике или известна
как малварь. **Ничего не изменяет и не удаляет** — только отчёт.

```bash
python -m malko scan-persist
python -m malko scan-persist --json persist.json
```

Замечание: на Windows с неанглийской локалью колонки вывода `schtasks`
локализованы — такие задачи пропускаются с пояснением в отчёте.

### kev-list — свежие записи CISA KEV

Выводит последние записи каталога известных эксплуатируемых уязвимостей CISA
(удобно для демонстрации). Лента кэшируется локально в `~/.malko/cache/kev.json`
на 24 часа.

```bash
python -m malko kev-list --since 2026-07-01
python -m malko kev-list --limit 5
```

## Защита в реальном времени

### monitor — сторож каталогов

Следит за указанными каталогами (рекурсивно): каждые `--interval` секунд
(по умолчанию 2) снимает «слепок» файлов и определяет созданные/изменённые/
удалённые. Каждый новый или изменённый файл хэшируется (SHA-256, потоковое
чтение) и сверяется с локальным блоклистом вредоносных хэшей. При совпадении
файл **немедленно перемещается в карантин**, выводится заметное предупреждение
и пишется строка в журнал `~/.malko/monitor.log`. Файлы больше `--max-size`
мегабайт (по умолчанию 100) пропускаются. Чистые файлы молчат, кроме режима
`--verbose`. Работает до Ctrl+C, при остановке печатает статистику.

```bash
python -m malko monitor C:\Users\me\Downloads
python -m malko monitor C:\Users\me\Downloads --proc --interval 1 --verbose
python -m malko monitor ~/Downloads --live-api   # неизвестные файлы — через live API (нужен MB_AUTH_KEY)
```

Флаг `--proc` добавляет мониторинг процессов: для каждого **нового** процесса
определяется путь к исполняемому файлу (Windows: wmic/PowerShell; Linux:
`/proc/<pid>/exe`), он хэшируется и сверяется с блоклистом. При совпадении —
громкое предупреждение в консоль и журнал. **Процессы не завершаются
автоматически** — только оповещение (убивать чужой процесс рискованно).
Неразрешимые пути (системные процессы, права) тихо пропускаются.

### blocklist-update — обновление блоклиста

Скачивает текстовый экспорт SHA-256-хэшей малвари MalwareBazaar
(`https://bazaar.abuse.ch/export/txt/sha256/recent/`, API-ключ не нужен) в
`~/.malko/cache/sha256-blocklist.txt`. Кэш действует 24 часа; `monitor`
обновляет его автоматически при запуске, если он устарел. Проверка членства —
мгновенная (хэши загружаются в множество).

```bash
python -m malko blocklist-update
```

Путь к файлу блоклиста можно переопределить переменной окружения
`MALKO_BLOCKLIST_PATH` (например, для офлайн-проверок со своим списком).

### quarantine — управление карантином

Вредоносные файлы перемещаются в `~/.malko/quarantine/` под сгенерированным
именем; рядом пишется sidecar-файл `<id>.json` (исходный путь, SHA-256, время,
источник: `monitor` или `scan-files`).

```bash
python -m malko quarantine list                 # список записей
python -m malko quarantine restore <id>         # вернуть файл на место
python -m malko quarantine restore <id> --force # перезаписать, если путь занят
```

`restore` отказывается перезаписывать существующий файл без `--force`.
Переменная `MALKO_QUARANTINE_DIR` переопределяет каталог карантина.

## Веб-панель

Встроенная веб-панель в тёмном «security console» стиле (русский интерфейс,
без внешних зависимостей и CDN — работает офлайн):

```bash
python -m malko dashboard                  # откроет браузер автоматически
python -m malko dashboard --port 8899 --no-browser
```

Панель слушает **только 127.0.0.1** — снаружи она недоступна. Что показывают
карточки и разделы:

- **Блоклист** — число хэшей и время последнего обновления;
- **счётчики** — проверено файлов, угроз найдено, перемещено в карантин,
  сводка последнего сканирования;
- **Журнал событий** — живой канал (автообновление каждые ~2 с): запуски
  монитора и сканирований, обновления блоклиста, тревоги;
- **Находки** — таблица: источник, бейдж серьёзности (CRITICAL — красный,
  HIGH — оранжевый, MEDIUM — жёлтый, LOW — синий, UNKNOWN — серый),
  CVE/хэш, флаг KEV;
- **Карантин** — список записей с кнопкой «Восстановить»;
- **Действия** — кнопки обновления блоклиста и запуска
  `scan-deps`/`scan-system`/`scan-files` по введённому пути (путь проверяется:
  только абсолютный и существующий; сканирования идут в фоне, статус — в
  журнале событий).

Данные панель читает из файлов состояния `~/.malko/state.json` (счётчики,
находки) и `~/.malko/events.jsonl` (события), которые ведут монитор,
сканеры, карантин и обновление блоклиста, — поэтому панель можно запустить
рядом с работающим `monitor` и видеть его тревоги в реальном времени.
Каталог состояния переопределяется переменной `MALKO_STATE_DIR`.

## Цветной терминал

В интерактивном терминале (Windows 10+, включён режим virtual terminal)
команды печатают ASCII-баннер MAL-KO и раскрашивают отчёты: бейджи
серьёзности по цветам, жирные заголовки, строка сводки — по цвету худшей
находки. При выводе в pipe или при установленной переменной `NO_COLOR`
оформление автоматически отключается и вывод остаётся чистым текстом.


## Формат результатов

Каждая сканирующая подкоманда печатает читаемую сводку (количество находок по
уровням серьёзности, затем список находок, отсортированный от CRITICAL к
UNKNOWN) и поддерживает флаг `--json <файл>` для машиночитаемого отчёта.
Код выхода: `0` — находок нет, `1` — есть находки, `2` — ошибка
(например, нет сети или не поддерживается система).

## Источники данных

| Источник | Назначение | URL |
|---|---|---|
| OSV.dev | уязвимости пакетов PyPI/npm | https://api.osv.dev/v1/querybatch |
| NVD (NIST) | CVE по ключевому слову | https://services.nvd.nist.gov/rest/json/cves/2.0 |
| CISA KEV | активно эксплуатируемые уязвимости | https://www.cisa.gov/sites/default/files/feeds/known_exploited_vulnerabilities.json |
| MalwareBazaar (abuse.ch) | хэши вредоносных файлов | https://mb-api.abuse.ch/api/v1/ |
| MalwareBazaar export | текстовый экспорт хэшей для блоклиста | https://bazaar.abuse.ch/export/txt/sha256/recent/ |

## Сборка standalone .exe

Можно собрать один самодостаточный исполняемый файл — на целевой машине
Python не нужен. Сборка использует PyInstaller, но ставит его **только в
локальное виртуальное окружение `.venv`** (зависимостей у самого mal-ko
по-прежнему нет):

```bash
python scripts/build_exe.py
```

Скрипт создаст `.venv`, установит туда PyInstaller и соберёт
`dist/mal-ko.exe` (печатает путь и размер).

**Запуск:** двойной клик по `mal-ko.exe` (или запуск без аргументов) открывает
веб-панель в браузере — консольное окно остаётся открытым и останавливается
по Ctrl+C; если порт 8888 занят, автоматически берётся ближайший свободный.
Режим командной строки — через аргументы, как обычно:

```bash
dist\mal-ko.exe --help
dist\mal-ko.exe monitor C:\Users\me\Downloads
dist\mal-ko.exe dashboard --port 9000
```

То же правило работает и при запуске через Python: `python -m malko` без
аргументов в интерактивном терминале открывает панель (при выводе в pipe —
прежнее сообщение об ошибке с подсказкой).

Замечания: для сборки нужен интернет (скачивание PyInstaller); антивирусы
иногда ругаются на onefile-сборки PyInstaller — известная ложная тревога на
сам упаковщик, при необходимости добавьте файл в исключения своего AV.
